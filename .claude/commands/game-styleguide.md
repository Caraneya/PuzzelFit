# GameStyleguide — Per-Game Visual Layer

Run after `/game-passport` is complete. Takes the game bible and Figma gameplay screen design, extracts game-specific design tokens, and generates `[game-name].css` and `[game-name]-styleguide.html`.

---

## Step 1 — Confirm Game

Ask: "Which game are we styling? I will read its game bible first."

Read `Games/[game-name]/[game-name]-game-bible.html`. Note:
- Game name and core mechanic
- Board/grid structure and dimensions
- Pieces/tiles/dice and their value range
- Game feel keywords
- Score display, timer presence, combo system

---

## Step 2 — Request Figma Gameplay Design

Say: "Share the Figma gameplay screen design. I will extract all game-specific colors, board styling, piece styling, and component exceptions from it."

Wait for the Figma URL or screenshot.

---

## Step 3 — Extract Game-Specific Tokens from Figma

Document only values that differ from or extend `../../tokens.css`. Skip anything that already matches a base token.

```css
/* Board and Grid */
--game-board-bg, --game-cell-empty, --game-cell-occupied, --game-cell-border, --game-board-radius

/* Pieces / Tiles / Dice */
--game-piece-1 through --game-piece-N, --game-piece-text, --game-piece-radius

/* Score and Timer */
--game-score-color, --game-score-bg, --game-timer-color, --game-timer-warn-color

/* Feedback and Effects */
--game-combo-color, --game-vanish-color, --game-merge-color
```

---

## Step 4 — Generate `Games/[game-name]/[game-name].css`

Replace the existing stub entirely.

```css
/* === [Game Name] — Game Stylesheet ===========================
   Per-game CSS overrides and gameplay-specific custom properties.
   Requires: ../../tokens.css, ../../components.css
   Generated by: /game-styleguide skill
   =========================================================== */

:root {
  /* ── BOARD ──────────────────────────────── */
  --game-board-bg: ...;

  /* ── PIECES ─────────────────────────────── */
  --game-piece-1: ...;

  /* ── SCORE AND TIMER ────────────────────── */
  --game-score-color: ...;

  /* ── FEEDBACK ───────────────────────────── */
  --game-combo-color: ...;
}

/* ── COMPONENT OVERRIDES ────────────────────────────────────
   Only add rules here if this game deviates from components.css.
   ─────────────────────────────────────────────────────────── */
```

STRICTLY ENFORCED:
- Only game-specific values — never repeat base token values from `../../tokens.css`
- All color values must reference CSS custom properties — zero hardcoded hex
- If a Figma hex has no matching base token, assign it to a `--game-*` variable first

---

## Step 5 — Generate `Games/[game-name]/[game-name]-styleguide.html`

Replace the existing stub entirely.

Link order: `../../tokens.css` → `../../components.css` → `./[game-name].css`
`data-theme="light"` on html element. `.back-btn` links to `../../GamePage.html`.

Sections:
1. **Color Palette** — every `--game-*` token as a swatch card (name + role label, fill via CSS variable)
2. **Board and Grid** — render actual board using game.css tokens, exact BOARD_COLS × BOARD_ROWS
3. **Piece / Tile / Die Examples** — each piece value with its `--game-piece-N` color
4. **Score and Timer** — styled with game tokens, show warning state
5. **Component Exceptions** — list every `components.css` class overridden; if none: "No component overrides."

Pipeline footer: GamePassport (done) → GameStyleguide (current) → GameBuild

---

## Handoff

```
Game Styleguide complete.
  Games/[game-name]/[game-name].css ([N] custom properties defined)
  Games/[game-name]/[game-name]-styleguide.html

Next step: /game-build
```
