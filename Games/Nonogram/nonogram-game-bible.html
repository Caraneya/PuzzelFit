Nonogram Game Bible · In Development
Open Flow Prototype
Primary Action

The player is presented with a grid and a set of number clues along every row and every column. To solve the puzzle, the player taps an empty cell to fill it black. A second tap on a filled cell marks it with an X — indicating it is definitely empty. A third tap resets it to blank. The goal is to fill the correct cells so that each row's filled sequences match the row clues, and each column's filled sequences match the column clues.

Match Rule

A row or column is considered solved when its consecutive groups of filled cells — reading left-to-right or top-to-bottom — exactly match the clue numbers in order. Groups must be separated by at least one empty cell. The puzzle is won when every row and every column is solved simultaneously. Tapping a cell that contradicts any clue costs a life and triggers an error animation on the affected row or column clue.

Cell States
Empty Default state. The cell has not been interacted with. Shown as a light square with a subtle border.
Filled The player has tapped this cell and believes it should be black. Shown as a solid dark square. First tap from empty.
✕
Marked Empty (X) The player is certain this cell must remain empty and places an X as a placeholder to avoid accidental fills. Second tap from filled, or long-press from empty. Does not cost a life.
Hinted Cell When the player uses a hint, one cell that can be logically deduced is briefly highlighted in accent color. The system uses the overlap method to identify the most constrained unsolved cell. After a moment the highlight fades and the cell is auto-filled.
Hint System

Hints are available via a button in the score bar. Each hint costs one hint charge (players start with 3 per game). The engine scans all unsolved rows and columns and applies the overlap method: for each clue number, it calculates the leftmost and rightmost valid positions. Any cell that falls in both positions is guaranteed filled — the hint reveals the highest-confidence cell. If no cell can be guaranteed, the hint instead reveals which row or column is most constrained, nudging the player to examine it.

Tutorial (5 steps)
1Complete rows: To start, choose the dark square. Color the rows based on the numbers nearby.
2Sequences: Each number represents a square sequence. If an end is already filled, there is only one solution.
3Space and order: More numbers indicates that there must be at least one space. The order indicates how the squares must be arranged.
4Overlaps: If the number sequences overlap in both the columns and the rows, a black square must be placed where they touch.
5Final steps: Check the boxes that are definitely empty. Not sure? Use the other color as a placeholder and replace it at the end.

Skill Expression

A beginner relies on trial and error — guessing cells and losing lives when wrong. An expert works deductively: using the overlap method to identify guaranteed cells first, then cross-referencing row and column constraints to eliminate possibilities without ever guessing. Experts mark definite empty cells with X to avoid accidental fills and solve complex puzzles with zero errors, unlocking perfect-solve bonuses and higher score multipliers.

Animation Mood

Playful but precise. Cell fills have a quick pop-scale (120 ms) that feels satisfying and snappy. Incorrect taps trigger a brief red flash and a horizontal shake on the violated clue number. Completing a row or column sends a warm golden sweep across the solved line and fades the clue number to a lighter tint. Solving the full puzzle triggers a cascade ripple from the last-placed cell, spreading outward through all cells before the win screen appears.

Engagement Loops
Core Loop · ~30 sec
Scan clues → deduce constrained cell → tap to fill (or mark X) → match check → correct: score pop + clue progress; incorrect: shake + life lost → repeat until puzzle solved or lives/timer exhausted.
Session Loop · 2–4 min
Start puzzle → early easy deductions build momentum → mid-game intersections create tension → final constrained cells ramp up → cascade win animation → results screen (score, errors, time bonus) → CTA: Play Again or Daily Challenge.
Meta Loop · Daily / Weekly
Daily streak → daily challenge modifier → personal best chase → share score card → cosmetic unlock (future) → leaderboard position → weekly reset challenge.

Retention hooks: daily puzzle resets at midnight, streak badge on stats screen, perfect-solve badge visible to friends, increasing score target for consecutive daily wins.

Daily Challenge Concept

One modifier per day changes the rules of a fixed shared puzzle (same grid worldwide). Examples: No Hints — hint charges removed entirely; Speed Run — timer halved, score doubled; No X Marks — placeholder marking disabled, test pure deduction; One Life — a single error ends the game; Blackout — clue numbers hidden after the first 30 seconds.
